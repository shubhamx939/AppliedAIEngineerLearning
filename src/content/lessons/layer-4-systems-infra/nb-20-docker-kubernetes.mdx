## What this lesson is really about

Containers and Kubernetes often feel abstract until you connect them to actual workloads: API servers, background workers, vector services, and inference runtimes with uneven resource needs.

This lesson focuses on how infra abstractions affect developer speed, operational safety, and workload placement for AI systems.

## What you will understand

- Understand what containers solve for AI services and what they do not.
- Read Kubernetes primitives through the lens of model workloads and APIs.
- Design more reliable deployment and rollout strategies.
- Think clearly about scaling, scheduling, and resource isolation.

## Why this topic matters in real AI engineering

Learn how containerization and orchestration shape the deployment story for AI APIs, retrieval services, and model workers. This lesson should help you notice which parts are conceptual, which parts are operational, and which parts are the usual source of failure when a team tries to move too quickly.

## How to use this lesson

Read the sections in order the first time. On later passes, use the subtopic map below as a reference spine: it is arranged to help you connect ideas rather than memorize isolated terms.

```dockerfile
WORKDIR /app
COPY pyproject.toml .
RUN pip install -U pip && pip install .
COPY . .
CMD ["uvicorn", "src.main:app", "--host", "0.0.0.0", "--port", "8000"]
```
